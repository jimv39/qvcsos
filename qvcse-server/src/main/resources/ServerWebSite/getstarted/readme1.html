QVCSOS Enterprise -- Installation and Getting Started...

Before you can use QVCSOS-Enterprise, you must first download and install an appropriate Java virtual machine. We have tested with Oracle's JVMs and OpenJDK VMs. Oracle's download site is available at this URL:

https://www.oracle.com/java/technologies/

We have tested with Java 17.

You also need to install the Postgres database. We have tested with Postgres 13. We've tested on MacOS and Ubuntu. In theory, things should work on any platform where you can install the needed tooling.

Build your own

QVCSOS-Enterprise is open source. You can build your own copy of the server and client applications. See https://github.com/jimv39/qvcsos for some guidance on getting started. After building the applications, you can relocate the created testenterprise directory tree to where ever makes sense for you.

Installing and Configuring Postgres

QVCS Enterprise uses a Postgres database to store your project history. You can download this open source database via a number of different methods. Do a web search to find an approach that works for you.

After installing Postgres, you need to create the database that QVCSOS Enterprise uses. See the SQL scripts located in the testenterprise directory. There are 2 different scripts in that directory: one for development use, and one for production use. They are the same, they just have different schema names, etc. Having some fluency in SQL/psql will help, though the SQL is not really that complex.

Once you have created the database, you need to edit the qvcsos.postgresql.connection.properties file (in the testenterprise/testDeploy/qvcsBehaviorProperties directory) to update the database connection information so that the server can connect to the Postgres database.
