General Questions

What is "version control"?

Wikipedia has a good background article on version control systems. In summary, version Control Systems and QVCS serve a number of purposes. These include:

  • Automate the tracking of files as they change during the course of a development project.
  • Provide the ability to recover previous revisions of files.
  • Allow multiple lines of development to proceed at the same time for a project.
  • Audit the development process so you can keep track of who changes which file, when they make the change, and what changes they make.
  • Reduce storage requirements.
  • Create reports to show what files have changed between product releases.

This list is not exhaustive, but it gives you a sample of the kinds of things that QVCSOS can do for you. QVCSOS lets you make changes to the files of your project with the confidence that you can go back to previous snapshots of those files if you need to.

How many separate files do you have under version control for managing the QVCSOS-Enterprise product itself?

There are 1172 files in the product, including Java source files, ant build scripts, and web site files.

Is there a version of QVCSOS-Enterprise available for platform fill-in-the-blank?

QVCSOS-Enterprise should work on any platform that has a Java 8 JVM available for it. QVCSOS is developed primarily on Ubuntu Linux, and tested on Linux and MacOS using Oracle's JVMs and OpenJDK.

QVCSOS-Enterprise Support Questions

Where do I find the client log file?

The client log file will be located in the logs directory beneath the directory containing the qvcse-gui-VERSION.jar file. The log file will have a name like qvcsos-client.log.

Where do I find the server log file?

The server log file will be located in the logs directory beneath the directory containing the qvcse-server-VERSION.jar file (where VERSION is the version of QVCSOS-Enterprise). The log file will have a name like qvcsos-server.log.

What files should I backup?

On the server:

On the server, you should backup the postgres database. You should perform server backups on a regular basis.

On the clients:

On the client, you may want to backup the client install directory (the one containing the qvcse-gui-VERSION.jar file) and all its sub-directories. Backup on the client is not so critical as server backups, in that data on the client can be regenerated without too much effort.

I'm getting 'out of memory' exceptions on the server. What should I do?

  • Shutdown the server by typing CTRL-C in the command window where the server is running, or by using the admin tool to shutdown the server.
  • Edit the server.sh file, adding -Xmx512M before the -jar on the command line used to start the server.
  • Restart the server.

I'm getting 'out of memory' exceptions on the client. What should I do?

Try this:

  • Exit the client application.
  • Edit the client.sh file, adding -Xmx512M before the -jar on the command line used to start the client.
  • Restart the client.
