Ignoring files on the client

On the client, you can create a .qvcsosignore file in the root directory of a project that the client application will use to identify files in that project that the client should ignore.

Unlike git, there can only be a single .qvcsosignore file for a project. It is located in the project root directory on the client, and there is a separate one for each project. If no .qvcsosignore file exists for a project, then none of the files in the project tree will be ignored. (Note there is a separate user preference that allows the client to ignore so-called hidden files.)

The .qvcsosignore file has no effect to the server. Files that are ignored may still be under source control. User A's .qvcsosignore file may be completely different than User B's.

A sample .qvcsosignore explained

1:  /logs/
2:  target/
3:  /.git/
4:  qvcs.served.project.Test Project.properties
5:  wip.odt
6:  /copy.xml
7:  /testenterprise/version.sh
8:  pom.xml

Line 1 /logs/ causes the log directory tree located at the root of the project to be ignored.
Line 2 target/ causes any target directory tree located anywhere in the project's directory tree to be ignored.
Line 3 /.git/ causes the .git directory tree located at the root of the project to be ignored.
Line 4 qvcs.served.project.Test Project.properties causes the file qvcs.served.project.Test Project.properties to be ignored anywhere in the project directory tree.
Line 5 wip.odt causes the file wip.odt to be ignored anywhere in the project directory tree.
Line 6 /copy.xml causes the file copy.xml located in the root directory of the project to be ignored.
Line 7 /testenterprise/version.sh causes the file version.sh located in the /testenterprise project directory to be ignored.
Line 8 pom.xml causes all pom.xml files to be ignored, irrespective of their location in the project directory tree.

Some observations:

  • If an entry begins with the / character, then the entry is anchored to the root of the project directory tree.
  • If an entry ends with the / character, then the entry refers to a directory, and its children
  • If the entry neither begins with a / or ends with a /, then the entry refers to a single file.
