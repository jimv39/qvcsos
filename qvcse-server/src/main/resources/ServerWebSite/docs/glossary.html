Glossary

branchA branch is a view of a project. A project has at least one branch (the TRUNK branch). Each branch has a separate location for that branch's workfiles. Typically, branches are used to isolate different lines of development. For example, user A might be working to create feature A while user B might be working on feature B. They could create separate branches for their work, and then promote their features to a parent branch when they finished their respective work.
tagA tag is just a string that you use to mark some point of development for a given branch. Typically, you apply/create a tag for a branch when you want to identify some significant point of progress for the given branch... maybe to define a release, or the completion of some feature, etc. Tags are durable, and just capture the commit id at the time of their creation.
tip revisionThe newest revision of a file.
TRUNKThe default branch for a project. When a project is first defined, the TRUNK branch is created automatically.
workfileThe file that you work with. Generally, a workfile is a text file, or source code, or other file that you edit or change using the tool of your choice. QVCSOS-Enterprise makes snapshots of the workfile contents.
